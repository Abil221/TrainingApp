# 🔧 Инструкция по применению миграции для целей, планов и достижений

## ⚠️ Проблема
Таблицы `user_goals`, `workout_plans`, `weight_history`, `user_levels`, `achievements` и `user_achievements` не были созданы в вашей Supabase базе данных, поэтому цели и планы тренировок не сохраняются.

## ✅ Решение

### Способ 1: Через Supabase Dashboard (Рекомендуется)

1. **Откройте Supabase Dashboard**
   - Перейдите на https://app.supabase.com
   - Выберите ваш проект

2. **Откройте SQL Editor**
   - В левом меню нажмите "SQL Editor"
   - Нажмите "New query"

3. **Скопируйте SQL код**
   - Откройте файл: `supabase/migrations_goals_plans.sql`
   - Скопируйте весь текст

4. **Выполните SQL**
   - Вставьте код в SQL Editor
   - Нажмите "Run" или `Ctrl+Enter`
   - Дождитесь, пока выполнится (должно быть зелёное сообщение "Success")

5. **Проверьте таблицы**
   - Перейдите в "Table Editor" в левом меню
   - Должны появиться новые таблицы:
     - `user_goals`
     - `weight_history`
     - `user_levels`
     - `achievements`
     - `user_achievements`
     - `workout_plans`
     - `workout_plan_days`

### Способ 2: Через Supabase CLI (если установлен)

```bash
# В папке проекта
supabase db push

# или вручную
psql "postgresql://..." < supabase/migrations_goals_plans.sql
```

## 🔒 Безопасность (RLS)

Все таблицы имеют Row Level Security (RLS) политики, которые гарантируют, что:
- Пользователи видят только свои данные
- Пользователи не могут видеть данные других пользователей
- Пользователи могут только читать и редактировать свои записи

## ❓ Возможные ошибки

### Ошибка: "relation already exists"
- Таблица уже создана ❌
- **Решение:** Это нормально, миграция содержит `IF NOT EXISTS`, так что повторный запуск безопасен

### Ошибка: "permission denied"
- У аноникусного ключа нет прав 🔐
- **Решение:** Убедитесь, что используете сервис ключ (service_role) или правильный aнонимный ключ с верными RLS политиками

### Ошибка: "column "id" does not exist"
- Таблица `profiles` не была создана
- **Решение:** Сначала запустите `supabase/schema.sql`
